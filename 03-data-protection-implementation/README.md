# 📋 Programa de Protección de Datos Personales – Caso Banco Demo S.A.

## 📌 Contexto del proyecto (caso ficticio)

Este proyecto demuestra mi experiencia como **Delegado de Protección de Datos (DPD)** en una entidad financiera simulada denominada **Banco Demo S.A.** (nombre completamente ficticio). El objetivo fue diseñar e implementar un programa integral de protección de datos personales alineado con la normativa ecuatoriana (LOPDP) y estándares internacionales.

El programa incluye el diagnóstico de brechas, la estructura de gobernanza, la definición de roles y responsabilidades (matriz RASCI), un plan de implementación estratégico, y un roadmap con fases, plazos e hitos.

## 🛠️ Metodologías y marcos aplicados

- **LOPDP (Ley Orgánica de Protección de Datos Personales – Ecuador)** – Base legal.
- **ISO 27701** – Extensiones de privacidad para ISO 27001.
- **ISO 27001** – Gestión de seguridad de la información.
- **MAGERIT v3.0** – Metodología de análisis y gestión de riesgos (aplicada a datos personales).
- **NIST CSF** – Marco de ciberseguridad para gobierno y mejora continua.

## 🎯 Resultados clave del proyecto

- Diagnóstico de brechas estructurado y priorizado.
- Definición del modelo de gobernanza con independencia del DPD.
- Recomendación de contratación de proveedor externo especializado para tareas operativas.
- Plan de implementación con ejes estratégicos, acciones priorizadas y plazos.
- Matriz RASCI que protege al DPD de funciones operativas.
- Roadmap con fases claras y criterios de éxito.
